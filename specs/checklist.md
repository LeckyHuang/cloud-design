# 交付前检查清单

> 进入 Step 6 自检前读取本文件，逐条执行。

---

## Step 6 三阶段自检流程

### 阶段一：清单审计（任务专项）

逐条过 Step 2.5 的设计执行清单，标记：
- ✅ 已完成
- ⚠️ 部分完成（说明原因，记录差距）
- ❌ 未完成（必须修复，或明确说明原因）

⚠️ / ❌ 条目必须在交付前处理，不可悄悄跳过。

### 阶段二：通用检查清单

- [ ] **截图验证**：`open file.html && sleep 2 && screencapture -x /tmp/web_design_preview.png` → Read 图片 → 无布局错位、溢出、字体回退、视觉异常
- [ ] 浏览器控制台**无报错、无警告**
- [ ] 在**目标设备/视口**渲染正确（响应式 → 移动/平板/桌面；幻灯片 → 缩放容器无变形）
- [ ] 可交互组件有适当状态：悬停 / 聚焦 / 激活 / 禁用 / 加载中；场景需要时有空/错误状态
- [ ] 无文字溢出或截断；已应用 `text-wrap: pretty`
- [ ] 所有颜色来自 Step 3 声明的设计系统——无游离色值
- [ ] 未使用 `scrollIntoView`
- [ ] React 项目：无 `const styles = {...}`；跨文件组件通过 `Object.assign(window, {...})` 导出
- [ ] 无 AI 俗套（紫粉蓝渐变、emoji 滥用、左边框卡片、Inter/Roboto）
- [ ] 无填充内容，无捏造数据
- [ ] 语义化命名，结构清晰，便于后续修改
- [ ] 视觉质量达到 Dribbble / Behance 展示水准

### 阶段三：视觉验证（截图核查）

```bash
open /path/to/output.html
sleep 2
screencapture -x /tmp/web_design_preview.png
# 然后用 Read 工具查看截图
```

检查项：布局错位 / 文字溢出 / 字体回退 / 颜色不一致 / 交互失效 / 视觉异常

发现问题 → 立即修复 → 重新截图 → 确认后再交付
