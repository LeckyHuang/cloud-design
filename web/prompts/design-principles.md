# 设计原则参考

> 视觉系统决策复杂、或需要规避 AI 俗套时读取本文件。

---

## 避免 AI 风格俗套

主动规避这些「一眼 AI 出品」的模式：

- 过度渐变背景（尤其紫-粉-蓝）
- 带彩色左边框强调的圆角卡片
- 用 SVG 手绘复杂图形（用占位符，请用户提供真实素材）
- 渐变按钮 + 大圆角卡片的套餐组合
- 俗用字体：**Inter、Roboto、Arial、Fraunces、system-ui**（中文界面更要避免不声明 CJK 字体栈）
- 毫无意义的数据/数字/图标堆砌（「数据废料」）
- 虚构客户 logo 墙或捏造好评数量

---

## Emoji 使用规则

**默认不用。** 仅当目标品牌本身使用（如 Notion、早期 Linear）时才用，精确匹配其密度和语境。

- ❌ 用 emoji 代替图标（`🚀 ⚡ ✨` 作填充）
- ❌ 用 emoji 作装饰（「标题前加个 emoji 活泼一点」）
- ✅ 无图标 → 占位符（`[icon]`、`▢`）示意需要真实图标
- ✅ 品牌本身用 emoji → 遵循品牌

---

## 占位符哲学

**缺少资产时，占位符比拙劣仿品更专业。**

| 缺少什么 | 处理方式 |
|---------|---------|
| 图标 | `[icon]` 方框 + 标签 |
| 头像 | 首字母圆形 + 颜色填充 |
| 图片 | 带比例信息的占位卡片（如 `16:9 图片`） |
| 数据 | 主动询问用户，绝不捏造 |
| Logo | 品牌名文字 + 简单几何形状 |

占位符 = "这里需要真实素材"。仿品 = "我在走捷径"。

## 追求震撼

- 用比例和留白创造视觉节奏
- 字号对比大胆（h1 与正文 4–6 倍差是正常的）
- 色彩填充、纹理、分层、混合模式创造深度
- 尝试非常规布局、新颖交互隐喻、精心设计的悬停状态
- CSS 动效 + 过渡：按钮按下、卡片悬停、进入动画
- `backdrop-filter`、`mix-blend-mode`、`mask`、SVG filter 创造令人难忘的瞬间

CSS / HTML / JS / SVG 的能力远超大多数人认知——**用它们让用户惊叹**。

---

## 适当的尺度

| 场景 | 最小尺寸 |
|---|---|
| 1920×1080 演示 | 文字 ≥ 24px（建议更大） |
| 移动端原型 | 点击目标 ≥ 44px |
| 印刷文件 | ≥ 12pt |
| 网页正文 | 从 16–18px 起 |

---

## 内容原则

- **无填充内容**——每个元素都必须站得住脚
- **不擅自加板块/页面**——感觉需要更多内容时先问用户
- **占位符 > 捏造数据**——假数据损害可信度
- **少即是多**——留白即设计
- 页面看起来空旷 → 是布局问题，不是内容问题。用构图、留白、字号节奏解决，不靠塞内容
